## 4. Cross-cutting insight (the "nobody wrote this" idea)

**The vault keeps rebuilding the same two primitives:**
1. An **event-driven, idempotent state machine** (seen in 02's baseline, QRIS orchestrator, logistics orchestrator) — stable, copy-pasteable.
2. An **anchor-of-trust registry** — every trust/scam/compliance product needs *one canonical "is this safe/valid/compliant?" lookup* but each doc builds its own isolated list:
   - judol-pinjol → OJK whitelist + Satgas PASTI blocklist
   - scam-detection → Kominfo blocklist + OSINT
   - desil/dormant checker → SIKS-NG desil + bank dormant status
   - MBG compliance → BGN SPPG compliance status
   - QRIS/credit scoring → acquirer reconciliation

If the vault ever ships these as products, they should share **one registry service** (or one MCP `lookup_trust(entity)` tool). Writing `anchor-of-trust-registry.md` now (new gap #4) plants the flag before the four products diverge.

Secondary insight: **the 01 crawler already has the query grammar to *populate* every one of these registries** (regulatory-shock pattern, early-panic pattern). The missing link is a dedup/canonicalization layer so the same pain surfacing 5×/week collapses into one signal — a prerequisite for any registry.
