# _templates/ — canonical file shapes

Copy the matching template when creating a new file. They mirror the real formats already
in the vault (demand-mining pains, bottlenecks, competitors, opportunities, inbox seeds,
MCP specs, pulse JSON).

| Template | Use for | Target folder |
|----------|---------|---------------|
| [`demand-mining.md`](demand-mining.md) | a mined pain (≥3 sources, strength, wedge) | `03/.../demand-mining/` + log in `INDEX.md` |
| [`bottleneck.md`](bottleneck.md) | deep structural analysis w/ code | `03/.../bottlenecks/` |
| [`competitor.md`](competitor.md) | competitor / platform gap teardown | `03/.../competitors/` |
| [`opportunity.md`](opportunity.md) | build-ready one-pager | `07/opportunities/` |
| [`inbox-seed.md`](inbox-seed.md) | short seed idea | `07/inbox/` |
| [`mcp-spec.md`](mcp-spec.md) | agent / MCP server spec | `04/.../mcp-servers/` |
| [`pulse-data.json`](pulse-data.json) | live feed snapshot schema | `05/data/`, `06/data/` |

Conventions (naming, headers, data contracts, credential hygiene) → [`../_meta/CONVENTIONS.md`](../_meta/CONVENTIONS.md).
